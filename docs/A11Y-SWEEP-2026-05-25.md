## Fixes applied this commit

### 1. Resources page — selects had no programmatic label

**File:** `src/pages/Resources.tsx`

The "Domain" and "Resource type" `<select>` controls had visible `<label>` siblings but no `htmlFor`/`id` association and no `aria-label`. Screen readers couldn't associate the label with the control (WCAG 1.3.1 + 4.1.2 violation).

**Fix:** added `aria-label="Filter resources by domain"` and `aria-label="Filter resources by type"` directly on the selects. The visible `<label>` text remains for sighted users.

### 2. PageLoader — no live-region announcement

**File:** `src/App.tsx`

The lazy-route loading placeholder (`<div>Loading…</div>`) rendered without `aria-live`, so AT users got no announcement when navigation triggered a chunk load. Page transitions appeared silent.

**Fix:** added `role="status"` + `aria-live="polite"` so the "Loading…" string is announced once on each lazy route load.

## What this audit did NOT cover (follow-up items)

These each require either a running browser, an external tool, or device hands-on. The architecture-agent ranked Lighthouse + axe-DevTools as the right next pass.

### High value, low effort

1. **Lighthouse pass** against the production URL — capture A11y + Performance + SEO + Best Practices scores. Set as a baseline; treat any future PR that drops scores as a regression.
2. **axe-DevTools pass** across all 16 routes — surfaces contrast failures, focus-order issues, and dynamic-content gaps that static audit can't see.
3. **Keyboard-only walkthrough** — tab through every route, confirm focus moves in DOM order, confirm focus visible on dark-theme backgrounds. Particular care: SignalDetail collapsible sections, AcknowledgmentModal close/Escape, mobile nav menu toggle.

### Medium value

4. **Color-contrast audit on muted text** — `var(--color-text-muted)` against `var(--color-bg-primary)` likely sits near the 4.5:1 WCAG AA threshold on small text. Verify on deployed site; bump if it fails. Affects the `0.625rem` uppercase labels used heavily on Overview/Status/Resources.
5. **Reduced-motion preference** (`prefers-reduced-motion`) — no current animations are likely to cause vestibular harm, but if any future component adds parallax/auto-rotate, add the media-query guard.
6. **Screen-reader smoke test** — one pass through the dashboard with VoiceOver (macOS Safari) or NVDA (Windows Firefox) on the Overview + SignalDetail + News routes. Validates landmark/heading navigation works in practice.

### Lower priority

7. **`<select>` styling on Resources page** — visible labels are styled as sibling `<label>` elements without `htmlFor`. The `aria-label` fix above closes the SR gap, but a future refactor could associate them properly via `id`/`htmlFor` so click-on-label focuses the select (UX improvement, not strict a11y).
8. **NetworkPage graph alternative** — the SVG is labeled `role="img"` + `aria-label="Signal relationship network graph"`, which satisfies the WCAG requirement, but doesn't expose the underlying data structure to AT users. A linked `<table>` summary of the same relationships would let SR users access the network content. Future enhancement.
9. **Map page (Leaflet)** — Leaflet panels are known to be a11y-hostile out of the box. The dashboard's map is read-only and the same data is reachable via Signals + Timeline routes (which are fully a11y-clean), so this is informational only.
