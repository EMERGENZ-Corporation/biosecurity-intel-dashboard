## System Summary

Static Vite/React public health dashboard for the 2026 Andes hantavirus outbreak. GitHub Actions is the autonomous data writer; Vercel serves the static dashboard. Runtime API routes were removed because the app does not depend on serverless API calls.

## Current State

- Automated data update workflow runs every 6 hours and can be manually dispatched.
- Official CDC, ECDC, WHO, and PHAC pages are fetched and parsed directly, with Bright Data fallback where configured.
- Gemini extraction is optional for enrichment. If Gemini is unavailable or rate-limited, the pipeline falls back to capped RSS-only updates instead of failing the run.
- Data validation checks JSON structure, metric provenance, manual overrides, official source health, and status endpoint consistency.
- Parser regression tests cover official-source extraction cases.
- Production verification checks that both the deployed app bundle and `/status.json` contain the expected fresh timestamps.
- Public `status.json` is generated by the updater and exposes status, thresholds, data freshness, source freshness, pipeline failures, and runbook guidance.
- Independent hourly production status monitor watches `/status.json` and reuses a `status-monitor` GitHub issue for alerts.
- `npm audit` reports 0 vulnerabilities after removing unused Vercel API dependencies.

## Decisions Made

- Treat noncritical RSS feed failures as degraded telemetry, not fatal pipeline failures.
- Treat critical feed plus critical official-source failure during an active outbreak as fatal.
- Keep Google News fallback capped and disabled by default through `FALLBACK_GOOGLE_NEWS_LIMIT=0`.
- Use `public/status.json` as the monitorable health contract instead of scraping the UI.
- Avoid committing or modifying the existing untracked `PROJECT-SNAPSHOT.md`.
- Keep the dashboard as a static Vite deployment with no runtime serverless API dependency.
